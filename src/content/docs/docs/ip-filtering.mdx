The **`[security.ip_filter]`** block controls which client IPs may connect, using IPv4/IPv6 **CIDR** ranges. **Dynamic** — changes apply on reload without restarting the process.

There is **no** GeoIP or ASN filtering in the proxy itself.

## Modes

- **`disabled`:** no IP-based blocking (default in many examples).
- **`allowlist`:** only listed networks can reach the proxy. An **empty** allowlist **denies all** traffic.
- **`denylist`:** listed networks are blocked. An **empty** denylist **allows all** traffic.

## Disabled (default in many examples)

<Tabs syncKey="huginn-config">
  <TabItem label="YAML">

```yaml
security:
  ip_filter:
    mode: "disabled"
```

  </TabItem>
  <TabItem label="TOML">

```toml
[security.ip_filter]
mode = "disabled"
```

  </TabItem>
</Tabs>

## Allowlist (only these networks can reach the proxy)

<Tabs syncKey="huginn-config">
  <TabItem label="YAML">

```yaml
security:
  ip_filter:
    mode: "allowlist"
    allowlist:
      - "127.0.0.1/32"
      - "::1/128"
      - "10.0.0.0/8"
```

  </TabItem>
  <TabItem label="TOML">

```toml
[security.ip_filter]
mode = "allowlist"
allowlist = [
  "127.0.0.1/32",
  "::1/128",
  "10.0.0.0/8",
]
```

  </TabItem>
</Tabs>

## Denylist (block specific ranges)

<Tabs syncKey="huginn-config">
  <TabItem label="YAML">

```yaml
security:
  ip_filter:
    mode: "denylist"
    denylist:
      - "198.51.100.0/24"
```

  </TabItem>
  <TabItem label="TOML">

```toml
[security.ip_filter]
mode = "denylist"
denylist = ["198.51.100.0/24"]
```

  </TabItem>
</Tabs>
